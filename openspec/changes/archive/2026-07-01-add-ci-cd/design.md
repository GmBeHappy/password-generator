## Architecture

Single GitHub Actions workflow at `.github/workflows/ci.yml`.

```
push to main / PR to main
  ↓
checkout + Setup Docker Buildx
  ↓
Login to GHCR (docker/login-action + GITHUB_TOKEN)
  ↓
Docker metadata (tags, labels from docker/metadata-action)
  ↓
Build + Push (docker/build-push-action, cache enabled)
  ↓
Output: ghcr.io/gmbehappy/password-generator:latest
         ghcr.io/gmbehappy/password-generator:sha-abc1234
```

## Workflow Structure

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

permissions:
  contents: read
  packages: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: docker/setup-buildx-action@v3
      - uses: docker/login-action@v3
      - uses: docker/metadata-action@v5
      - uses: docker/build-push-action@v6
```

## Action Selection

| Step | Action | Version | Rationale |
|---|---|---|---|
| Checkout | `actions/checkout` | v4 | Standard, fast |
| Buildx | `docker/setup-buildx-action` | v3 | Multi-platform support, cache |
| Login | `docker/login-action` | v3 | GHCR auth, `github.actor` + `GITHUB_TOKEN` |
| Metadata | `docker/metadata-action` | v5 | Auto-generate tags, labels, OCI annotations |
| Build+Push | `docker/build-push-action` | v6 | Build and push in one step, cache support |

## Tagging Strategy

Generated by `docker/metadata-action`:

```
type=raw,value=latest,enable={{is_default_branch}}
type=sha,prefix=sha-,format=short
```

Result:
- Push to main: `latest`, `sha-abc1234`
- PR: no push, build-only verification

## Cache Strategy

```yaml
cache-from: type=gha
cache-to: type=gha,mode=max
```

Uses GitHub Actions cache backend. Docker layers (deps, source, build artifacts) persist between workflow runs. Typical cache hit on repeated builds: ~80% of layers.

## Registry Path

- **Registry:** `ghcr.io`
- **Image:** `ghcr.io/gmbehappy/password-generator`
- Derived from `github.repository` to keep workflow portable

## Authentication

No secrets needed. Workflow uses built-in `secrets.GITHUB_TOKEN` with `packages: write` permission. The `docker/login-action` authenticates as the GitHub actor pushing the commit.

```yaml
with:
  registry: ghcr.io
  username: ${{ github.actor }}
  password: ${{ secrets.GITHUB_TOKEN }}
```
